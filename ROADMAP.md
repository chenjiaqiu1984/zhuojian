## 🔴 第一批：服务闭环（有了这些才有完整产品）

**目标**：让用户能完整体验"预约 → 咨询 → 付费 → 得到服务"的完整链路。

### 🎥 1. 视频 / 语音咨询（2-3周）

**为什么排第一**：没有视频咨询，用户预约完要跳到微信，咨询师可以绕开平台私下接单，平台无法沉淀服务。视频咨询是核心服务交付，不是体验提升。

- [ ] 技术选型：腾讯实时音视频 TRTC（推荐，与微信生态兼容，有 uni-app SDK）
- [ ] 后端集成 TRTC SDK，生成用户/咨询师房间 Token
- [ ] 新增咨询房间模型 `ConsultRoom`（bookingId、roomId、status、startTime、endTime、duration）
- [ ] 前端集成 TRTC uni-app SDK
- [ ] 预约时间到达时，双方各自收到"进入咨询室"入口
- [ ] 支持视频/语音切换、静音、摄像头开关
- [ ] 咨询结束后自动关闭房间，记录时长
- [ ] 异常断线重连机制（网络抖动时自动重连）
- [ ] 管理后台：咨询记录查看（时长、状态、异常）

**涉及文件**：
- 新增 `backend/routes/room.js`
- 新增 `backend/services/trtc.js`
- 新增前端页面 `frontend/src/pages/room/`
- 修改 `backend/routes/booking.js`（预约触发房间创建）

### 💳 2. 支付系统（2-3周，需与视频咨询并行）

> ⚠️ 注意：微信支付商户号审核需要1-3个工作日，要提前申请，不要等到开发完成再申请。

**前置条件**：先完成数据库迁移（见技术债务），再上线支付，避免SQLite写锁影响订单成功率。

- [ ] **提前申请**微信支付商户号（需营业执照）、支付宝商户号
- [ ] 数据库升级至 MySQL（在支付上线前完成，见技术债务）
- [ ] 后端新增订单模型 `Order`（orderNo、userId、amount、status、payType、refId、expireAt）
- [ ] 实现支付回调接口（微信/支付宝异步通知 + 签名验证）
- [ ] 咨询预约付费流程（预约时创建订单 → 15分钟未支付自动取消）
- [ ] 量表单次付费购买（与现有代金券逻辑并存）
- [ ] 退款接口（取消预约时根据距咨询时间自动计算退款比例）
- [ ] 管理后台：订单列表、退款操作、收入统计、咨询师结算

**涉及文件**：
- 新增 `backend/routes/payment.js`
- 新增 `backend/services/wechatpay.js`、`backend/services/alipay.js`
- 修改 `backend/prisma/schema.prisma`（新增 Order 模型）
- 修改 `backend/routes/booking.js`（预约时触发支付）

## 🟠 第二批：供给侧与留存

**目标**：让咨询师愿意留在平台，让用户有持续回来的理由。

### 👨‍⚕️ 4. 咨询师专属工作台（4-5天）

**为什么提前**：咨询师是平台供给侧。工作台体验差，优质咨询师会流失或不愿入驻，直接影响用户侧口碑。

- [ ] 咨询师独立登录入口（与用户端分离的路由/角色）
- [ ] 今日预约时间线（直观展示当天日程）
- [ ] 来访者档案：历史预约次数、做过的量表、咨询师私人备注（仅自己可见）
- [ ] 作业布置：向指定来访者推送量表或自助练习任务
- [ ] 消息中心：新预约通知、评价提醒、系统公告
- [ ] 收入统计：本月咨询收入、已结算/待结算金额

**涉及文件**：
- 新增 `frontend/src/pages/therapist/`
- 修改 `backend/routes/consultants.js`（工作台专用接口）
- 修改角色权限中间件（therapist角色）

## ⚙️ 技术债务（与业务迭代并行处理）

| 优先级 | 事项 | 时机 | 说明 |
|--------|------|------|------|
| 🔴 紧急 | 数据库迁移至 MySQL/PostgreSQL | **支付上线前** | SQLite写锁在并发场景会导致订单失败，不能等 |
| 🔴 紧急 | 图片迁移至腾讯云 COS + CDN | 第一批期间 | 本地存储无法扩容，服务器重装数据丢失 |
| 🟠 重要 | 接口文档（Apifox） | 第二批前 | 咨询师工作台接口多，无文档协作混乱 |
| 🟠 重要 | 错误监控（Sentry） | 支付上线前 | 支付相关异常必须第一时间发现 |
| 🟡 一般 | 后端核心路由单元测试 | 第二批期间 | 支付、预约、退款逻辑必须有测试覆盖 |
| 🟡 一般 | 性能监控 | 第二批期间 | 核心接口 P99 响应时间监控 |

## ❓ 待决策的产品问题（开发前需要确认）

在启动第一批开发之前，以下问题建议先想清楚，否则会影响技术实现：

1. **定价策略**：单次咨询多少钱？平台抽成比例？咨询师结算周期？
2. **咨询模式**：纯线上视频？还是也支持线下预约（平台只做引流）？
3. **咨询师入驻**：开放申请还是邀请制？资质审核标准是什么？
4. **冷启动**：第一批用户和咨询师从哪里来？有没有种子用户计划？
5. **差异化**：市面上已有简单心理、壹心理、好心情，卓见心理的核心差异是什么？OH卡牌 + 心理咨询的结合，还是其他？
